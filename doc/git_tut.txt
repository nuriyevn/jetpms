0. В нулевую очередь необходимо проверить настроен ли удлаенный репозиторий. Иными словами видит ли локальный репозиторий удаленный. Выполняем git remote -v. Как понять что локальный видит удаленный репозиторий? Если при выводе команды есть слова типа origin и ссылка на github - то вот он удаленный репозиторий. 
Глубокий философский смысл локального репозитория - этот то что это твоя песочника, твое лично рабочее пространство. 
Глубокий философский смысл удаленного репозитория - собирать наработки со всех локальных репозиториев (если есть что-то полезное). Степень полезности решает автор изменений и в каком-то смысле администратор репозитория который решает полезность автора.
Как настроить чтобы локальный репошник видел глобальный? 'git remote add'. Для того чтобы можно было коммитить и пушить необходимо знать КТО  пушит? Если учесть что локальных репозиториев  >  чем авторов, поскольку в репозитории пушат обычно несколько человек и при этом у одного человека может быть несколько репозиториев - на разных компьютерах, та даже на одном компе, выходит, что локальных репозиториев - туча. И вполне логично что репозиторию (что локальному , что глобальному нужно знать КТО является автором пушей), и вот тут-то на нужна команда 'git config'. Хотя и удаленных может быть несколько - но сейчас не об этом.
1. Шаг ноль нужно выполнить только один раз для каждого локального репозитория. Есть такое понятие как папка локального репозитория. Есть и другое понятие - как древовидная структура папок. Понятно дело что в папке нашего проекта - может быть сколь угодное количество вложенных папок. Так вот, у репозитория - есть корневая папка. Корневой папкой является та, в которой была выполнена команда 'git init'.  Внимание! Большинство git команд необходимо выполнять в корневой папке репозитория. 
Некоторые команды могут выполняться и в подпапках корневой папки, например git add, git status, возможно это не совсем то что хочет выполнить новичок, поэтому рекомендуется переходить в корневую папку репы и выполнять там соответствующие команды.

Что делать если ты вчера хорошенько отдохнул и уже не помнишь где эта чертова корневая директория репозитория и вообще ты полностью дезориентирован, не понимаешь где находишься? Есть два способа узнать. Если будучи в какой-то папке  утилита git-bash высвечивает в спобках (master) то тут есть какой-то репозиторий(какой - смотришь по содержанию файлов в папке).
Второй способ узнать - более ядерный и надежный, если ты суров и работаешь не из git bash а из классической консоли без этих блекшлюзов и джекблеков, без этих подсвечиваний и прочмх свистоперделок, то в таком случае набираем 'ls -l'   . Если при выводе данной команды есть папка .git  (начинается с точки) - то это папка какого-то репозитория. Внимание! если просто ввести ls  без флага -l  то папки и файлы начинающиеся с точки выводиться не будут. Файлы и папки - начинающиеся с точки в линуксе имеют глубокий философский смысл который заключается в том что этих папок и файлов при обычном выводе ls - не видно. Поэтому и нужен флаг -l команды ls.
